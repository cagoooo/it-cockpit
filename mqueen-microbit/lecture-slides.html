05 · MECHANIC / 機制 ②

組裝 + 馬達控制

兩輪速度差，就是轉彎的秘密。

🔧 組裝四步驟

  • 裝電池：3 顆 AAA(7號)，注意正負極
  • 插超音波：T 對 trig、E 對 echo
  • 插 micro:bit：金手指朝下、LED 朝前
  • 開電源：車尾開關切 ON ⚡

⚠ 插拔感測器前務必先切 OFF！

⚡ 馬達速度差 = 轉彎

左 100、右 100直走
左 0、右 100左轉
左 100、右 0右轉
左 200 後退、右 200 前進原地打轉
速度範圍0~255
💡 馬達校正小技巧
└── 如果小車直走會偏（兩顆馬達摩擦力不同），調整左右速度差修正。
    例：左 255、右 240 來補償右輪較滑。
🛠️
這 4 件事學完，你就會控制機器人的雙腿了。

06 · SENSOR / 循跡感測

沿著黑線跑的眼睛

兩顆紅外線眼睛
+ 如果 / 否則 = 自動駕駛雛型。

/* 黑線輸出 0、白色輸出 1 */

感測值判斷動作
L1=0 R1=0兩顆都壓黑全前進
L1=0 R1=1左壓黑、右偏白向左修正
L1=1 R1=0右壓黑、左偏白向右修正
L1=1 R1=1兩顆都白跑出黑線！

▼ 循跡程式積木

⚡ 重複無限次
🔀 如果 L1=0 且 R1=0
▶ 馬達全前進 100
否則如果 L1=0 且 R1=1
▶ 左輪停、右輪 100
否則如果 L1=1 且 R1=0
▶ 左輪 100、右輪停

📐 場地：白色圖畫紙 + 黑色絕緣膠帶

👀
這就是所有自動駕駛車的雛形：偵測 → 判斷 → 修正。

08 · BOSS / 警車挑戰

最終 Boss 戰

超音波避障 + 警報音效 + 紅藍車燈閃爍
= 智慧警車！

蝙蝠聲納原理
├── 發出 超音波（人耳聽不到）
├── 撞到障礙物 → 反彈
├── 計算「來回時間」 → 距離
└── < 30cm → 煞車轉彎！

⚠️ 常見坑：距離跳 0 公分

超音波偶爾會跳到 0（反射錯誤）。
解法：條件改成 distance ≠ 0 且 < 30，
並加 100ms 延遲防錯。

▼ 警車完整邏輯

⚡ 重複無限次
📐 distance = 讀取超音波
🔀 如果 distance < 30
▶ 馬達後退 + 隨機轉
🔊 播放警報音效
💡 紅藍 RGB 燈閃爍
否則
▶ 馬達全前進巡邏

🚓 跟特斯拉 AEB 自動煞車是同一個邏輯！

🚓
恭喜！做出來這個，你已經會寫自動駕駛的核心邏輯了。

09 · REMOTE / 無線遙控

紅外線 vs 藍牙

兩種遙控方式，各有應用情境。

紅外線遙控器（像電視）

方便、便宜、近距離

  • 按鍵代碼：2 前進、8 後退、4 左、6 右、5 停
  • 需設定紅外線接收引腳
  • 遙控器有 KEYES / NEC 兩種規格
  • 限制：必須對準感測器、會被遮擋
藍牙手機 / 平板 App

滑順、無方向限制

  • 平板下載 micro:bit 官方 App
  • 同按 A+B、按 Reset → 30 秒內配對
  • 加 Devices 擴展，寫 Gamepad 對應
  • 限制：配對麻煩、需要 App
💡 常見問題：按 6 卻往左轉？
└── 不同遙控器代碼不同（KEYES 與 NEC 解碼格式不同）。
    先寫一段「顯示 message」測試實際數字，再填正確指令。
📱
這就是「測試先於寫程式」的工程思維——先看再做。

10 · NOTES / 注意事項

過來人的血淚提醒

這 4 件事不注意，
會被機器人「氣到摔書」。

⚡

電源安全

插拔感測器前務必切 OFF，
避免短路燒板。
電池正負極不能反裝。

📥

下載時不拔線

背面黃燈閃時，
不可拔 USB 線。
等閃爍結束才安全。

📐

場地要平整

循跡場地：白色圖畫紙
+ 黑色絕緣膠帶。
地毯、瓷磚反光會影響。

🤝 同學間共享小技巧

  • 馬達會偏掉？校正左右輪速度差，不要硬撐
  • 程式有 Bug？用「顯示 message / 變數」debug，把感測值印出來看
  • 同學的小車跑得比較順？請他分享 .hex 檔，馬上抄一份
  • 遙控器規格不同？寫測試程式看實際代碼，5 分鐘解決
🧯
機器人會壞，但學到的邏輯思維永遠不會壞。
