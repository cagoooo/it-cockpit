STAGE 1

🧩 積木家族大集合

每種顏色的積木都有不同的超能力！先來認識它們

📖 Scratch 積木分類

Scratch 把所有積木依功能分成不同顏色，就像超級英雄團隊一樣，每個顏色負責不同的任務：

🔵 動作
Motion
讓角色移動、轉彎、滑行
🟣 外觀
Looks
改變造型、說話、顯示/隱藏
🩷 音效
Sound
播放聲音、改變音量音調
🟡 事件
Events
觸發程式的開關（點旗、按鍵）
🟠 控制
Control
重複、判斷、等待、分身
🔵 偵測
Sensing
碰到東西？按了什麼鍵？
🟢 運算
Operators
加減乘除、隨機數、比較大小
🟧 變數
Variables
記錄分數、生命值等數字
🩷 函式積木
My Blocks
自己定義新積木（進階）

👆 點擊每張卡片看說明！今天做遊戲會用到大部分的積木家族喔！

💡 記住口訣：藍色管移動、紫色管外觀、黃色管觸發、橘色管控制、淺藍管偵測、綠色管運算、深橘管變數！

Q2. 「如果⋯那麼」這種判斷積木是什麼顏色？

👩‍🏫 教師提示

建議先讓學生自由探索 Scratch 介面 3 分鐘，觀察左側積木區的顏色分類。可以請學生數一數總共有幾種顏色。強調「顏色 = 功能分類」的概念，不需要記住每個積木的用途，用到時自然會學會。

STAGE 2

🎭 準備遊戲角色與舞台

設計你的遊戲世界：背景、籃子和水果

🌐 Step A — 打開 Scratch

  1. 打開瀏覽器，前往 scratch.mit.edu，點選上方「創建」
  2. 你會看到一隻橘色的貓咪（小貓是 Scratch 的預設角色）。我們的遊戲不需要貓咪，先在右下角的角色清單中，對貓咪按右鍵 → 刪除

🎨 Step B — 選擇背景

  1. 點擊右下角「舞台」區域的「選個背景」按鈕（藍色山的圖示）
  2. 推薦選擇 Blue Sky 或 Colorful City（在「戶外」分類中找）
  3. 背景會立刻套用到舞台上，你可以隨時更換！

🧺 Step C — 加入籃子角色

  1. 點擊右下角「選個角色」按鈕（貓咪臉 + 加號圖示）
  2. 搜尋 「Bowl」 或 「Plate」（碗/盤子可以當籃子用）
  3. 找不到喜歡的？點「繪畫」自己畫一個籃子！用長方形 + 弧形組合
  4. 把角色名稱改成 Basket（在角色資訊區點名字就能改）

🍎 Step D — 加入水果角色

  1. 再按一次「選個角色」，搜尋 「Apple」 選一顆蘋果
  2. 把角色名稱改成 Fruit
  3. 想要多種水果？在「造型」分頁中，點「選個造型」搜尋 Bananas、Orange 加入更多造型！後面會用到
🎯 設計細節：確認角色大小適中 — 籃子大約佔畫面寬度的 1/6，水果比籃子小一點。可以在角色資訊的「大小」欄位調整（籃子建議 80-100，水果建議 50-70）。

Q1. 在 Scratch 中，遊戲的背景圖片叫什麼？

Q2. 一個角色可以有好幾種不同的「外型」，在 Scratch 中叫做？

👩‍🏫 教師提示

這個階段建議給學生 5-8 分鐘自由發揮設計角色和背景。可以鼓勵學生自己繪畫角色增加成就感。提醒學生先完成基本角色（籃子 + 1 種水果），進階造型可以最後再加。

STAGE 3

⌨️ 鍵盤操控籃子

用方向鍵讓籃子左右移動接水果！

🎯 目標：讓籃子用 ← → 方向鍵左右移動

先點選右下角角色清單中的 Basket（確認是籃子被選中，邊框會變藍色），然後照以下步驟組裝積木：

📦 BASKET 籃子的程式
當 ⚑ 被點擊
定位到 x: 0 y: -150
重複無限次
如果 按下 [右→] 鍵? 那麼
x 改變 (10)
如果 按下 [左←] 鍵? 那麼
x 改變 (-10)

🔧 組裝步驟（照順序做）

  1. 在左邊積木區點 事件（黃色），拖出 當 ⚑ 被點擊 到程式區
  2. 點 動作（藍色），拖出 定位到 x: 0 y: -150 接在下面。這讓籃子每次開始都在畫面下方中間
  3. 點 控制（橘色），拖出 重複無限次 接在下面。這讓程式一直不停檢查按鍵
  4. 從 控制 拖出 如果⋯那麼 放進「重複無限次」裡面
  5. 點 偵測（淺藍色），把 按下 [空白] 鍵? 拖到「如果」後面的六角形空格中，然後把 [空白] 改成 [右→]
  6. 從 動作 拖出 x 改變 (10) 放進第一個「如果⋯那麼」裡面
  7. 重複步驟 4-6 做第二個「如果⋯那麼」：偵測 [左←] 鍵，x 改變 (-10)
  8. 按綠色旗子 ⚑ 測試！籃子應該可以用 ← → 鍵左右移動了 🎉
💡 設計細節：x 改變 10 代表每次按鍵往右移 10 步。想讓籃子移動更快？把 10 改成 15 或 20 試試看！數字越大移動越快。
⚠️ 常見問題：如果按方向鍵沒反應，檢查你有沒有先點擊舞台（讓 Scratch 視窗取得焦點），然後再按方向鍵。

Q1. 想讓角色一直不停地檢查按鍵，要用哪個積木？

Q2. x 改變 (-10) 會讓角色往哪個方向移動？

👩‍🏫 教師提示

此階段引入「事件驅動」概念 — 程式不斷檢查條件（有沒有按鍵），有的話就執行動作。這是遊戲程式的核心邏輯。可以請學生嘗試把 10 改成不同數字（5、20、50），體會「參數」對行為的影響。y:-150 是舞台底部附近，可讓學生理解座標系統（x 左右、y 上下）。

STAGE 4

🍎 水果從天而降！

用「分身」積木讓水果不斷出現、往下掉

💡 什麼是「分身」？

想像你是影分身之術的忍者 🥷 — 你只有一個本體，但可以製造出很多分身。在 Scratch 中，建立 [自己] 的分身 就是讓一個角色不斷複製自己！

我們的水果角色（Fruit）只需要一個，但用分身就能讓畫面上同時出現很多顆水果 🍎🍎🍎

📦 水果的程式（共兩段）

先在右下角點選 Fruit 角色，然後組裝以下兩段程式：

📦 程式一：不斷製造分身
當 ⚑ 被點擊
隱藏
重複無限次
建立 [自己] 的分身
等待 (2) 秒
📦 程式二：每個分身的行為
當分身產生
顯示
定位到 x: 隨機取數 (-200) 到 (200) y: (170)
重複無限次
y 改變 (-3)
如果 ( y 座標 ) < (-170) 那麼
分身刪除

🔧 組裝步驟

  1. 程式一：拖出 當 ⚑ 被點擊，接上 隱藏（紫色外觀區），再接 重複無限次
  2. 在重複裡面放 建立 [自己] 的分身（橘色控制區），再放 等待 (2) 秒（每 2 秒產生一顆新水果）
  3. 程式二：另外拖出一個 當分身產生（橘色帽子形狀），這是第二段獨立的程式
  4. 接上 顯示，再接 定位到 x: __ y: __
  5. x 的位置用 隨機取數 (-200) 到 (200)（從綠色運算區拖進去），y 輸入 170（畫面最上方）
  6. 接上 重複無限次，裡面放 y 改變 (-3)（讓水果一直往下掉）
  7. 再放一個 如果⋯那麼，條件是 ( y 座標 ) < (-170)，裡面放 分身刪除（掉到最底就消失）
  8. 按 ⚑ 測試！你應該看到水果不斷從上方隨機位置往下掉 🍎⬇️
💡 設計細節：等待秒數控制水果出現的頻率（2 秒一顆）。y 改變 -3 控制掉落速度。後面的 STAGE 6 會教你怎麼讓速度越來越快！
🎨 進階挑戰：在「當分身產生」後面加一行 造型換成 [隨機]，這樣每顆分身水果會隨機出現不同造型（蘋果、香蕉、橘子）！需要先在「造型」分頁加入多種水果造型。

Q1. 「建立自己的分身」積木是什麼顏色？

Q2.「隨機取數 (-200) 到 (200)」的功能是什麼？

👩‍🏫 教師提示

「分身」是本課最重要的概念。可以用「影印機」比喻：本體是原稿（隱藏起來不動），分身是影印出來的副本（出現在隨機位置然後往下掉）。常見錯誤：學生忘記在程式一加「隱藏」，導致本體也出現在畫面上。提醒：程式一控制「工廠」（不斷產出分身），程式二控制「每個產品」的行為。

STAGE 5

⭐ 碰撞偵測與計分系統

碰到籃子就加分！建立分數和生命值

💡 什麼是「變數」？

變數就像一個可以記住數字的小盒子📦。遊戲裡的「分數」和「生命」都需要用變數來記錄。盒子上面寫著名字（例如「分數」），裡面放著一個數字（例如 0），你可以隨時改變裡面的數字。

🔧 Step A — 建立變數

  1. 點左邊積木區的 變數（深橘色分類）
  2. 點「建立一個變數」按鈕，輸入名稱 分數，選「適用於所有角色」，按確定
  3. 再建一個變數，名稱 生命
  4. 舞台左上角會自動出現「分數」和「生命」的顯示！

Q1. 「變數 [分數] 改變 (1)」這個積木的意思是？

Q2. 要偵測水果有沒有碰到籃子，用的是哪個積木？

👩‍🏫 教師提示

「設為」vs「改變」是學生最容易搞混的：「設為 0」是直接把盒子裡的數字換成 0（通常用在初始化）；「改變 1」是在現有數字上加 1。可以用「計算機清除 C 鍵 vs + 鍵」來比喻。建議讓學生玩一下計分模擬器理解概念。注意碰撞偵測的積木名稱必須和角色名稱完全一致（Basket），大小寫也要對。

✨ Step B — 加入特效（選做）

  1. 接到水果時，在 播放音效 下面加 圖像效果 [顏色] 改變 (25)（籃子會閃光！需要在 Basket 角色中用廣播觸發）
  2. Game Over 時，在 停止 [全部] 前面加 說出 [Game Over! 🎮] (2) 秒（水果角色顯示 Game Over 訊息）
  3. 進階挑戰 — 速度加快：建立一個新變數 速度，初始值設為 3。把 y 改變 (-3) 改成 y 改變 (0) - (速度)。再加一個 如果 (分數) 除以 (10) 的餘數 = (0)，就 變數 [速度] 改變 (1)（每得 10 分加快一點！）

🚀 Step C — 發表你的作品

  1. 點上方「檔案」→「儲存到電腦」先存一份備份（.sb3 檔案）
  2. 如果有 Scratch 帳號：點右上角「分享」按鈕 → 輸入專案名稱和說明 → 按「分享」
  3. 分享後你會得到一個網址，把它貼到班級群組讓同學玩你做的遊戲！🎉
  4. 讓同學玩完後給你回饋：太難還是太簡單？分數容易拿嗎？根據回饋調整數字（速度、等待秒數、生命數）

✅ 遊戲測試清單

按 ⚑ 開始測試，每項通過就打勾：

  • 按 ← → 鍵，籃子可以左右移動
  • 水果從上方不同位置掉下來
  • 水果碰到籃子會消失，分數 +1
  • 水果掉到底部會消失，生命 -1
  • 接到水果有音效
  • 生命歸零時遊戲停止
  • 再按一次 ⚑ 可以重新開始（分數歸零、生命回到 3）

❓ 常見問題 FAQ

Q. 水果只掉一次就不掉了？
檢查程式一有沒有用 重複無限次 包住「建立分身」和「等待 2 秒」。如果忘了包，就只會產生一個分身。另外確認「等待秒數」不要太大（建議 1-3 秒）。
Q. 水果碰到籃子但分數沒有加？
確認 碰到 [Basket]? 的名字和你的籃子角色名稱完全一樣（包含大小寫）。如果你的角色叫 basket（小寫），偵測裡也要選 basket。建議統一使用 Basket（大寫 B）。
Q. 遊戲結束後按 ⚑ 重新開始，分數沒有歸零？
檢查 Basket 籃子的程式最前面有沒有 變數 [分數] 設為 (0) 和 變數 [生命] 設為 (3)。每次按 ⚑ 時這兩行會把變數重新歸零。
Q. 按方向鍵但籃子不動？
先點一下 Scratch 舞台畫面（讓視窗取得焦點），再按方向鍵。也要確認你選的是 Basket 角色再寫程式。最後檢查 重複無限次 有沒有把兩個「如果」都包在裡面。
Q. 我想讓遊戲越來越難，怎麼做？
建立一個「速度」變數，初始值 3。把 y 改變 (-3) 改成 y 改變 (0 - 速度)。然後在分數每增加 10 分時，讓速度 +1。也可以縮短「等待秒數」讓水果出現得更頻繁。
