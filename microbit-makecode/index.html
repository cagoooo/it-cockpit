STAGE 01

認識 micro:bit 硬體 — 火柴盒大小的迷你電腦

micro:bit 是英國 BBC 為 5 歲以上孩子設計的微型電腦。它雖然只有火柴盒大小，卻內建了處理器、感測器、25 顆 LED、無線電 — 把它想像成一個「迷你大腦」，你教它做什麼，它就會做！

正面有什麼？

  • 25 顆 LED 矩陣（5×5）：紅色 LED 燈，可顯示文字、數字、圖案；同時也能當光感測器，測量周圍亮度。
  • A、B 按鈕：可單獨按、也可同時按（A+B），用來控制程式。
  • V2 金色 Logo：第三顆觸控感應器（手指碰就觸發）。
  • V2 麥克風：偵測音量、感應掌聲。

背面有什麼？

  • 處理器：micro:bit 的「大腦」，負責執行你寫的程式。
  • 加速度感測器：三軸偵測，可感知搖晃、傾斜、自由落體（玩骰子靠它）。
  • 指南針 / 磁力感測器：偵測方向、磁場強度。
  • 溫度感測器：測量處理器附近的溫度。
  • V2 喇叭：內建發聲裝置，可放音樂。
  • 無線電 / 藍牙：可與其他 micro:bit 通訊，或連手機平板。
  • Reset 重設按鈕：當機時重新啟動。

底部金手指與電源

  • 0、1、2 號腳位：GPIO，可用鱷魚夾接耳機、外接 LED、馬達。
  • 3V：3 伏特電源輸出。GND：接地（完成電路用）。
  • microUSB：連電腦傳程式 + 順便供電。
  • 電池座：接 2 顆 AAA 電池，可離開電腦獨立運作。
💡 V1 vs V2 怎麼分？翻過板子看右下角，印著「micro:bit V1.x」或「V2.x」。V2 多了麥克風、喇叭、觸控金 Logo。若用 V2 程式（如聽聲音）燒到 V1 板，會顯示哭臉 + 錯誤碼 927。

STAGE 02

MakeCode 啟程 — 認識編輯器與積木抽屜

MakeCode 是微軟為 micro:bit 設計的免費線上編輯器，不用註冊帳號，打開瀏覽器就能寫程式。寫完按下載得到 .hex 檔，拖到 micro:bit 就會跑！

打開 MakeCode 三步驟

  • 打開瀏覽器（建議 Chrome），輸入 makecode.microbit.org
  • 點「+ 新增專案」，幫專案取個英文名字（例如 my-badge）
  • 右上角齒輪 → Language → 中文（繁體），介面立刻變中文

認識主畫面三大區塊

  • 🟦 左側模擬器：不用接實機就能測試，可模擬搖動、按鈕、傾斜。
  • 🟧 中央積木抽屜：10 大分類（基本、輸入、音樂、LED、迴圈、邏輯、變數、數學、廣播、進階）。
  • 🟩 右側工作區：拖積木組合程式的「畫布」，可像拼圖一樣拼起來。

10 大積木分類 — 顏色就是分類！

  • 基本（藍）：顯示文字、圖示、暫停
  • 輸入（紫）：按鈕、搖晃、感測器
  • 音樂（桃紅）：播放音調、旋律
  • LED（黃）：控制 25 顆 LED 亮滅
  • 迴圈（綠）：重複執行
  • 邏輯（藍綠）：如果 ... 那麼 ... 否則
  • 變數（橘）：記錄數值（像記分板）
  • 數學（深藍）：加減乘除、隨機
  • 廣播 Radio（紅）：兩塊 micro:bit 無線通訊
  • 進階（灰）：函式、陣列、引腳、控制（高階用）
🎯 兩種事件積木最重要：「當啟動時」電源開只執行一次；「重複無限次」電源開著一直跑。所有指令積木都要放進事件積木的 C 型開口裡，孤零零放空白處不會跑！

STAGE 03

姓名徽章 + 笑臉 — 第一個程式 5 分鐘速成

用 2 個積木就能完成你的第一個程式！讓 micro:bit 顯示你的英文名字，再加一個大大的笑臉。

程式邏輯（拼成這樣就對了）

  • 把「重複無限次」（工作區已自動有）保留
  • 從「基本」抽屜拖「顯示文字」放進「重複無限次」C 型開口
  • 點字串改成自己的名字，例如 Hello! KAI
  • 再拖「顯示圖示」放在「顯示文字」下方
  • 下拉選單選「😊 笑臉」
  • 左側模擬器自動播放預覽，看到名字和笑臉輪流出現！
💡 為什麼名字「跑」很正常？因為 25 顆 LED 一次顯示不下英文字，會自動「滾動」（scroll）。如果你想要每個字停久一點，可以加「暫停 1000 毫秒」（基本抽屜）。

STAGE 04

搖一搖電子骰子 — 學會變數、隨機、條件判斷

把 micro:bit 變成一顆電子骰子！搖一搖，LED 隨機顯示 1～6 點。這關會用到變數（記分板）+ 隨機（亂數）+ 條件判斷（如果那麼）。

程式邏輯（4 個步驟）

  • 從「輸入」抽屜拖「當姿勢 晃動」放進工作區
  • 從「變數」抽屜點「建立一個變數」→ 取名 roll
  • 「變數」→「變數 roll 設為 0」拖進「當晃動」C 型開口
  • 「數學」→「隨機取數 0 到 10」放在 0 的位置，把 10 改成 6
  • 「邏輯」→「如果 ... 那麼 ... 否則」拖在變數設定下面
  • 「邏輯」→ 比較積木 = 放進「如果」空格
  • 「變數」→ roll + 數字 1 放兩邊
  • 「基本」→「顯示指示燈」放「那麼」開口，點中間一格代表 1 點
  • 點「+」加更多條件，依此類推到 6 點
  • 命名 dice → 下載 → 燒錄 → 搖搖看！
🧠 變數是什麼？變數就像籃球比賽旁邊的「記分板」，可以隨時擦掉改寫數字。我們建立 roll 來記住骰子點數。隨機取數就像在抽號碼球，每次抽出來都不一樣！

STAGE 05

A/B 按鈕剪刀石頭布 — 學會事件驅動

把 micro:bit 變成猜拳機！按 A 出剪刀、按 B 出石頭、A+B 同時按出布。這關學「事件驅動」— 有事情發生才做動作。

程式邏輯（3 個事件積木）

  • 「輸入」→「當按鈕 A 被按下」放進工作區
  • 「基本」→「顯示指示燈」放裡面 → 自畫剪刀（兩支斜線）
  • 重複上面動作換成「按鈕 B」→ 畫石頭（實心方塊）
  • 「輸入」→「當按鈕 A+B 被按下」→ 畫布（5×5 全亮）
  • 下載 rps.hex → 燒錄 → 跟同學猜拳大戰！
🎮 事件驅動是什麼？就像家裡的「防盜鬧鐘」平常安靜，「當有人開門」才會逼逼叫。MakeCode 的「當按鈕 A 被按下」就是事件積木 — 平常不做事，按下去才執行裡面的指令。
🚀 進階挑戰：學會 STAGE 5 後，可以試試「廣播 Radio」讓兩塊 micro:bit 互通訊。A 同學出剪刀，無線傳送 → B 同學的板子顯示結果（贏 / 輸 / 平手）！

⚡ 下載 .hex 並燒錄到 micro:bit

完整燒錄流程

  • 用 USB 線把 micro:bit 連到電腦（確認線是「資料傳輸線」，不是純充電線）
  • 檢查檔案總管出現 MICROBIT (D:) 磁碟機
  • MakeCode 左下角輸入專案名 → 按紫色「下載」按鈕
  • 檔案存到電腦「下載」資料夾 → 產生 microbit-XXX.hex
  • 把 .hex 檔拖到 MICROBIT(D:) 磁碟機（拖曳即可）
  • 背面黃燈閃爍 → 表示正在燒錄
  • 黃燈熄滅 + 磁碟重新整理 → 完成！程式自動執行
⚠️ 常見錯誤：找不到 MICROBIT 磁碟機 — 可能是 USB 線只能充電 / 不小心按到背面 Reset 進入 MAINTENANCE 模式 / USB port 接觸不良。
解法：換線、換 port，或請老師更新韌體。
💾 學校電腦小提醒：很多學校電腦關機後會自動還原，.hex 檔會被清掉！下課前一定要把 .hex 檔存到自己的 USB 隨身碟，下次用「匯入」讀回來繼續做。

▶ 學生最常問的 5 個問題

為什麼我寫好程式按下載，卻找不到 MICROBIT 磁碟機？
兩個可能：USB 線只能充電不能傳資料（換條線）；或插線時按到背面 Reset 變成 MAINTENANCE 磁碟機（請老師幫忙下載韌體更新檔拖進去就好）。
為什麼模擬器跑得動，真正的 micro:bit 卻沒反應？
按「下載」只是把 .hex 檔存到電腦「下載」資料夾，還沒送到 micro:bit。要打開檔案總管，把 .hex 檔拖到 MICROBIT 磁碟機，看到背面黃燈閃爍才是真的燒錄。
為什麼我的積木拼不上去，旁邊還變灰色？
MakeCode 的指令積木不能孤零零放空白處。要把它拖進有 C 型開口的「事件積木」裡（像「重複無限次」、「當按鈕被按下」），聽到「喀」一聲卡進去才算對！
我可以下次接著做嗎？檔案存在哪裡？
MakeCode 自動存在瀏覽器，但學校電腦有還原軟體關機後會消失。下課前一定要按「儲存」下載 .hex 檔，存到自己的 USB 隨身碟，下次用「匯入」讀回來。
micro:bit 沒接電池盒，怎麼還會亮燈？
USB 線有兩個任務：傳程式 + 順便供電給 micro:bit。所以接電腦時不用電池盒。要拔下來帶到操場或當穿戴裝置，才需要接電池盒（2 顆 AAA 電池）。
